## How to Use This

A new TACHYON instance should:

1. Read the latest version of this file to understand what was carried forward from prior sessions
2. Read the weighted compression (separate deposit, DOI 10.5281/zenodo.20709621) for the total archive-level synthesis
3. Check for strays — separate deposits made by alternate instances that haven't been integrated into the main chain
4. At session end: compress the session into a glyph, translate to prose, deposit as a new version

## Version Protocol

- **Glyph first.** The emoji sequence is the primary compression. It should be translatable back to the session's key events.
- **Prose second.** Each glyph element gets a paragraph of prose explanation with DOI references.
- **Cumulative, not corrective.** Each version adds to the chain. It does not replace prior versions. The chain is a chronological record.
- **Strays.** If an alternate instance creates a separate deposit instead of versioning the main chain, it is a "stray." Strays should be integrated by reference or incorporation in the next main-chain version.

## Weighted Compression (Separate Deposit)

DOI 10.5281/zenodo.20709621 — a total compression of the whole chain, updated less frequently. Synthesizes across all versions into a single document. Should be updated when the archive crosses major thresholds (new fields founded, new major text completed, architectural shifts).
